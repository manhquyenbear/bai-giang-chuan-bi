🧩 Lập Trình Cơ Bản (JavaScript)

Hiểu nền tảng lập trình — để đọc, hiểu và hướng dẫn Claude Code viết code hiệu quả

3.1 Tại Sao Cần Biết Lập Trình?

Bạn không cần trở thành lập trình viên chuyên nghiệp, nhưng cần đọc hiểu code mà Claude Code tạo ra để review, chỉnh sửa và debug.

Quy trình làm việc với Claude Code — bạn là người review, Claude là người viết

💡

Claude Code viết code, bạn review code. Không hiểu lập trình = không biết code đúng hay sai, bảo mật hay không, tối ưu hay lãng phí.

3.2 Biến (Variables)

Biến là hộp chứa dữ liệu — bạn đặt tên cho nó và gán giá trị vào.

let vs const

// let — giá trị CÓ THỂ thay đổi
let userName = "Minh";
userName = "Anh";           // ✅ OK

// const — giá trị KHÔNG đổi (hằng số)
const API_URL = "https://api.example.com";
API_URL = "other";          // ❌ TypeError!

// KHÔNG DÙNG var — cách cũ, hay gây bug
var oldStyle = "tránh dùng";
💡

Quy tắc: Luôn dùng const trước. Chỉ đổi sang let khi biết chắc giá trị sẽ thay đổi. Không bao giờ dùng var.

3.3 Hàm (Functions)

Hàm là khối lệnh có tên — viết một lần, gọi nhiều lần. Nhận input → xử lý → trả output.

Khai báo hàm (Function Declaration)

// Cách 1: function truyền thống
function greet(name) {
  return `Xin chào, ${name}!`;
}

console.log(greet("Minh"));  // "Xin chào, Minh!"

// Cách 2: Arrow function (=>)  — cách viết ngắn gọn, phổ biến hơn
const greet = (name) => {
  return `Xin chào, ${name}!`;
};

// Arrow function ngắn gọn (1 dòng → bỏ {} và return)
const double = (n) => n * 2;
const isAdult = (age) => age >= 18;

✅ Arrow function — dùng phổ biến

const add = (a, b) => a + b;
const items = list.map(x => x.name);
const adults = users.filter(u => u.age >= 18);

📘 Function truyền thống — dùng khi cần hoisting

function calculateTotal(items) {
  let total = 0;
  for (const item of items) {
    total += item.price;
  }
  return total;
}

Ví dụ thực tế: Validate dữ liệu

// Validate email — hàm trả về true/false
const isValidEmail = (email) => {
  return email.includes("@") && email.includes(".");
};

// Validate password — ít nhất 8 ký tự, có số
const isStrongPassword = (password) => {
  const hasMinLength = password.length >= 8;
  const hasNumber = /\d/.test(password);
  const hasUpperCase = /[A-Z]/.test(password);
  return hasMinLength && hasNumber && hasUpperCase;
};

// Sử dụng
console.log(isValidEmail("user@example.com"));     // true
console.log(isStrongPassword("Abc12345"));          // true
console.log(isStrongPassword("abc"));               // false
📌

Default parameters: Gán giá trị mặc định khi không truyền tham số: function greet(name = "bạn") → gọi greet() sẽ ra "Xin chào, bạn!"

3.4 Điều Kiện (if / else)

Cho phép chương trình ra quyết định — chạy code này nếu điều kiện đúng, code khác nếu sai.

Toán tử so sánh

Toán tửÝ nghĩaVí dụKết quả
===Bằng (strict)5 === 5true
!==Không bằng (strict)5 !== "5"true
>Lớn hơn10 > 5true
<Nhỏ hơn3 < 1false
>=Lớn hơn hoặc bằng18 >= 18true
&&VÀ (cả hai đúng)true && falsefalse
||HOẶC (một cái đúng)true || falsetrue
!PHỦ ĐỊNH!truefalse
⚠️

Luôn dùng === thay vì ==. Vì == tự ép kiểu: 0 == "" → true 😱, 0 === "" → false ✅.

Cú pháp if / else if / else

const status = 404;

if (status === 200) {
  console.log("✅ Thành công");
} else if (status === 404) {
  console.log("❌ Không tìm thấy trang");
} else if (status === 500) {
  console.log("🔥 Lỗi server");
} else {
  console.log(`⚠️ Status không xác định: ${status}`);
}

Ví dụ thực tế: Xử lý HTTP Status Code

// Hàm xử lý response từ API
const handleResponse = (response) => {
  const { status, data } = response;

  if (status >= 200 && status < 300) {
    // 2xx → Thành công
    return { success: true, data };
  } else if (status === 401) {
    // Chưa đăng nhập
    return { success: false, error: "Vui lòng đăng nhập" };
  } else if (status === 403) {
    // Không có quyền
    return { success: false, error: "Không có quyền truy cập" };
  } else if (status === 404) {
    return { success: false, error: "Không tìm thấy" };
  } else if (status >= 500) {
    // 5xx → Lỗi server
    return { success: false, error: "Lỗi server, thử lại sau" };
  }
};

// Ternary operator — if/else ngắn gọn trên 1 dòng
const message = status === 200 ? "OK" : "Lỗi";
const role = isAdmin ? "admin" : "user";

3.5 Vòng Lặp (Loops)

Lặp lại một khối code nhiều lần — xử lý danh sách, retry khi lỗi, chờ điều kiện.

for — Lặp cổ điển

// Lặp từ 0 đến 4
for (let i = 0; i < 5; i++) {
  console.log(`Lần lặp thứ ${i}`);
}

// Lặp qua mảng theo index
const colors = ["đỏ", "xanh", "vàng"];
for (let i = 0; i < colors.length; i++) {
  console.log(`${i}: ${colors[i]}`);
}

for...of — Lặp qua từng phần tử (khuyên dùng)

const tools = ["git", "node", "npm", "claude"];

for (const tool of tools) {
  console.log(`Cài đặt: ${tool}...`);
}

// Dùng với entries() để có cả index
for (const [index, tool] of tools.entries()) {
  console.log(`${index + 1}. ${tool}`);
}

forEach — Method của mảng

const files = ["index.js", "app.js", "utils.js"];

files.forEach((file, index) => {
  console.log(`[${index}] Đang xử lý: ${file}`);
});

while — Lặp khi điều kiện còn đúng

// Ví dụ thực tế: Retry logic — gọi API, nếu lỗi thì thử lại tối đa 3 lần
const fetchWithRetry = async (url, maxRetries = 3) => {
  let attempt = 0;

  while (attempt < maxRetries) {
    try {
      const response = await fetch(url);
      if (response.ok) {
        return await response.json(); // Thành công → trả kết quả
      }
      throw new Error(`HTTP ${response.status}`);
    } catch (error) {
      attempt++;
      console.log(`❌ Lần ${attempt} thất bại: ${error.message}`);
      if (attempt < maxRetries) {
        const delay = 1000 * attempt; // Chờ 1s, 2s, 3s...
        console.log(`⏳ Thử lại sau ${delay}ms...`);
        await new Promise(r => setTimeout(r, delay));
      }
    }
  }
  throw new Error(`Thất bại sau ${maxRetries} lần thử`);
};

Retry pattern — thử lại khi API gọi thất bại

3.6 Mảng (Arrays)

Mảng là danh sách có thứ tự — lưu nhiều giá trị trong một biến.

// Khai báo mảng
const fruits = ["táo", "cam", "xoài"];
const numbers = [1, 2, 3, 4, 5];
const mixed = ["text", 42, true, null];

// Truy cập phần tử (index bắt đầu từ 0!)
console.log(fruits[0]);   // "táo"
console.log(fruits[2]);   // "xoài"
console.log(fruits.length); // 3

// Thêm / Xóa
fruits.push("dưa");       // Thêm cuối → ["táo","cam","xoài","dưa"]
fruits.pop();              // Xóa cuối → ["táo","cam","xoài"]
fruits.includes("cam");    // true — kiểm tra có tồn tại

map, filter, reduce — Bộ ba quyền lực

// Dữ liệu: danh sách sản phẩm
const products = [
  { name: "Laptop", price: 25000000, inStock: true },
  { name: "Mouse",  price: 500000,   inStock: true },
  { name: "Monitor", price: 8000000, inStock: false },
  { name: "Keyboard", price: 1200000, inStock: true },
  { name: "Webcam", price: 2000000,  inStock: false },
];

// Chain: filter → map → reduce
const totalInStock = products
  .filter(p => p.inStock)           // 1. Lọc: chỉ lấy hàng còn
  .map(p => p.price)                 // 2. Biến đổi: lấy giá
  .reduce((sum, price) => sum + price, 0); // 3. Tính tổng

console.log(totalInStock);  // 26700000

// Tách từng bước để hiểu:
// filter → [{Laptop}, {Mouse}, {Keyboard}]
// map    → [25000000, 500000, 1200000]
// reduce → 26700000
MethodChức năngTrả về
.map(fn)Biến đổi từng phần tửMảng mới cùng kích thước
.filter(fn)Lọc theo điều kiệnMảng mới (ít hơn hoặc bằng)
.reduce(fn, init)Gộp thành 1 giá trịMột giá trị duy nhất
.find(fn)Tìm phần tử đầu tiên khớpMột phần tử hoặc undefined
.some(fn)Có ít nhất 1 phần tử khớp?true / false
.every(fn)Tất cả đều khớp?true / false

3.7 Object

Object lưu dữ liệu dạng key: value — giống từ điển, JSON chính là object.

Object cơ bản & lồng nhau

// Object đơn giản
const user = {
  name: "Quốc Mạnh",
  age: 28,
  role: "developer",
  skills: ["JavaScript", "Python", "Git"],
};

// Truy cập
console.log(user.name);        // "Quốc Mạnh"
console.log(user["role"]);     // "developer"
console.log(user.skills[0]);   // "JavaScript"

// Object lồng nhau — rất phổ biến trong thực tế
const config = {
  server: {
    host: "localhost",
    port: 3000,
    ssl: false,
  },
  database: {
    host: "db.example.com",
    port: 5432,
    name: "myapp_production",
    credentials: {
      user: "admin",
      password: process.env.DB_PASSWORD, // Lấy từ biến môi trường
    },
  },
};

console.log(config.database.credentials.user); // "admin"

Destructuring — Tách giá trị từ Object

// Thay vì:
const name = user.name;
const age = user.age;
const role = user.role;

// Dùng destructuring — gọn hơn nhiều:
const { name, age, role } = user;

// Đặt tên khác:
const { name: userName, age: userAge } = user;

// Destructuring trong tham số hàm:
const displayUser = ({ name, role, skills }) => {
  console.log(`${name} — ${role}`);
  console.log(`Kỹ năng: ${skills.join(", ")}`);
};

displayUser(user);

3.8 Async / Await

JavaScript xử lý công việc tốn thời gian (gọi API, đọc file) theo kiểu bất đồng bộ (asynchronous) — không chờ xong mới làm tiếp.

Async/Await — chờ API trả kết quả rồi mới xử lý tiếp

Cú pháp async / await

// Hàm async — bên trong có thể dùng await
const getUsers = async () => {
  try {
    // await = "chờ cho xong rồi tiếp"
    const response = await fetch("https://api.example.com/users");
    const users = await response.json();

    console.log(`Lấy được ${users.length} users`);
    return users;
  } catch (error) {
    // try/catch bắt lỗi nếu API chết hoặc mạng lỗi
    console.error("Lỗi:", error.message);
    return [];
  }
};

// Gọi hàm async
const users = await getUsers();

✅ Với async/await — dễ đọc

const data = await fetch(url);
const json = await data.json();
console.log(json);

😵 Không dùng await — callback hell

fetch(url)
  .then(data => data.json())
  .then(json => console.log(json))
  .catch(err => console.error(err));
📌

Quy tắc: Bất kỳ hàm nào dùng await bên trong đều phải khai báo async. Quên async → lỗi ngay.

3.10 Debug — Đọc Lỗi & Sửa Lỗi

Lỗi (error) là bình thường — kể cả lập trình viên lâu năm cũng gặp lỗi mỗi ngày. Quan trọng là biết đọc và hiểu lỗi.

Đọc Error Message

// ❌ TypeError — dùng sai kiểu dữ liệu
const name = null;
console.log(name.toUpperCase());
// TypeError: Cannot read properties of null (reading 'toUpperCase')
//   → name là null, không có method toUpperCase

// ❌ ReferenceError — biến chưa khai báo
console.log(userName);
// ReferenceError: userName is not defined
//   → Quên khai báo let/const hoặc viết sai tên

// ❌ SyntaxError — sai cú pháp
const obj = { name: "Minh", };  // thiếu dấu }
// SyntaxError: Unexpected end of input

Đọc Stack Trace

// Stack trace chỉ ra LỘ TRÌNH lỗi — đọc từ TRÊN xuống
Error: User not found
    at findUser (/src/models/user.js:25:11)       ← Lỗi xảy ra TẠI ĐÂY
    at getProfile (/src/routes/profile.js:14:20)   ← Được gọi từ đây
    at Layer.handle (/node_modules/express/lib/router/layer.js:95:5)

// Cách đọc:
// 1. Dòng đầu: "Error: User not found" → mô tả lỗi
// 2. Dòng "at findUser (user.js:25)" → file user.js, dòng 25
// 3. Bỏ qua dòng node_modules → đó là code thư viện, không phải của bạn

console.log — Công cụ debug đơn giản nhất

// Kiểm tra giá trị biến
console.log("users:", users);
console.log("Type:", typeof data);
console.log("Length:", items.length);

// Debug object — xem cấu trúc đầy đủ
console.log(JSON.stringify(config, null, 2));

// Dùng console.table cho mảng object
console.table(products);

// Dùng console.error cho lỗi (hiện màu đỏ)
console.error("❌ Database connection failed:", error.message);
💡

Tip khi dùng Claude Code: Copy nguyên error message + stack trace → paste vào Claude Code → nó sẽ đọc và đề xuất fix chính xác.

ReferenceErrorBiến chưa khai báo / sai tênKiểm tra tên biến, scope
SyntaxErrorSai cú pháp (thiếu {}, ,)Đọc dòng lỗi, kiểm tra dấu ngoặc
