4.1 JSON Là Gì?

JSON (JavaScript Object Notation) là định dạng lưu trữ và trao đổi dữ liệu dạng text. Dễ đọc cho người, dễ parse cho máy.

4.4 Lỗi JSON Thường Gặp

JSON rất nghiêm ngặt — một lỗi nhỏ sẽ khiến toàn bộ file không parse được.

❌ Trailing Comma (Dấu phẩy thừa)

❌ Sai — dấu phẩy cuối

{
  "name": "project",
  "version": "1.0",  ← dấu phẩy thừa!
}

✅ Đúng — không có dấu phẩy cuối

{
  "name": "project",
  "version": "1.0"
}

❌ Single Quotes (Ngoặc đơn)

❌ Sai — ngoặc đơn

{
  'name': 'project',
  'active': true
}

✅ Đúng — ngoặc kép

{
  "name": "project",
  "active": true
}

❌ Comments (Bình luận)

❌ Sai — có comment

{
  // Tên project
  "name": "demo",
  /* Version hiện tại */
  "version": "2.0"
}

✅ Đúng — không comment

{
  "name": "demo",
  "version": "2.0"
}
💡

Mẹo: Dùng cat file.json | python3 -m json.tool để kiểm tra JSON hợp lệ. Trong VS Code, cài extension "JSON" sẽ highlight lỗi ngay.

4.5 YAML Là Gì?

YAML (YAML Ain't Markup Language) là định dạng cấu hình dựa trên thụt lề (indentation), dễ đọc hơn JSON cho người.

💡

YAML dùng ở đâu? GitHub Actions, Docker Compose, Kubernetes, CI/CD pipelines, frontmatter trong Markdown (SKILL.md, AGENTS.md), và cấu hình nhiều tool hiện đại.

# Cùng một dữ liệu — JSON vs YAML

# JSON (24 dòng)
{
  "server": {
    "host": "localhost",
    "port": 3000,
    "debug": true
  },
  "database": {
    "url": "postgres://localhost:5432/mydb",
    "pool": 10
  }
}

# YAML (7 dòng) — gọn hơn nhiều!
server:
  host: localhost
  port: 3000
  debug: true
database:
  url: postgres://localhost:5432/mydb
  pool: 10

4.6 Cú Pháp YAML

Key-Value cơ bản

name: My Project
version: 2.1.0
description: "Dự án sử dụng Claude Code"   # String có dấu đặc biệt cần ngoặc
active: true                                 # Boolean
count: 42                                    # Number
nothing: null                                # Null

Nested (lồng nhau) — dùng thụt lề 2 spaces

server:
  host: localhost
  port: 3000
  ssl:
    enabled: true
    cert: /path/to/cert.pem
    key: /path/to/key.pem

Arrays (Danh sách)

# Dạng block (mỗi item 1 dòng)
fruits:
  - apple
  - banana
  - cherry

# Dạng inline (1 dòng)
colors: [red, green, blue]

# Array of objects
users:
  - name: Alice
    role: admin
  - name: Bob
    role: developer

Multiline Strings

# | (Literal) — giữ nguyên xuống dòng
description: |
  Đây là dòng 1
  Đây là dòng 2
  Mỗi dòng giữ nguyên

# > (Folded) — gộp thành 1 dòng
summary: >
  Đây là một đoạn dài
  sẽ được nối thành
  một dòng duy nhất

Kiểu dữ liệu YAML

KiểuVí dụGhi chú
Stringname: helloKhông cần ngoặc (trừ ký tự đặc biệt)
Numberport: 3000Tự nhận diện
Booleandebug: truetrue/false, yes/no, on/off
Nullvalue: null hoặc value:Để trống cũng là null
Datedate: 2026-04-14ISO 8601 tự nhận diện
List- item1Dấu - ở đầu dòng

4.7 YAML Trong Claude Code

SKILL.md Frontmatter

Các file SKILL.md dùng YAML frontmatter (phần giữa ---) để khai báo metadata:

---
name: code-review
description: "Skill review code tự động"
version: 1.0
triggers:
  - "review this code"
  - "check for bugs"
tags:
  - review
  - quality
---

# Code Review Skill

Khi được yêu cầu review code, hãy:
1. Kiểm tra logic errors
2. Tìm security vulnerabilities
3. Đề xuất performance improvements

Agent Definition File

---
name: backend-agent
description: "Agent chuyên xử lý backend"
model: claude-sonnet
tools:
  - Read
  - Write
  - Bash

  allow:
    - "Bash(npm *)"
    - "Bash(node *)"
  deny:
    - "Bash(rm -rf *)"

  - src/server/

Slash Command (Custom)

# File: .claude/commands/deploy.md
---
name: deploy
description: "Deploy ứng dụng lên staging"
arguments:
  - name: environment
    description: "Môi trường deploy"
    required: true
    options:
      - staging
      - production
---

Hãy thực hiện deploy ứng dụng lên môi trường $environment:
1. Chạy tests
2. Build project
3. Deploy lên server

4.8 YAML cho GitHub Actions

GitHub Actions là nơi bạn gặp YAML nhiều nhất trong thực tế.

# .github/workflows/ci.yml
name: CI Pipeline                     # Tên workflow

on:                                   # Khi nào chạy?
  push:
    branches: [main, develop]         # Push vào main hoặc develop
  pull_request:
    branches: [main]                  # Tạo PR vào main

jobs:                                 # Danh sách jobs
  test:                               # Job tên "test"
    runs-on: ubuntu-latest            # Chạy trên Ubuntu
    strategy:
      matrix:
        node-version: [18, 20, 22]    # Test trên 3 version Node

    steps:                            # Các bước thực hiện
      - uses: actions/checkout@v4     # Checkout code

      - name: Setup Node.js           # Cài Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}

      - name: Install dependencies    # Cài packages
        run: npm ci

      - name: Run tests               # Chạy test
        run: npm test

      - name: Run lint                 # Kiểm tra code style
        run: npm run lint

  deploy:                             # Job tên "deploy"
    needs: test                       # Chỉ chạy sau khi test pass
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'  # Chỉ deploy từ main

    steps:
      - uses: actions/checkout@v4
      - name: Deploy to production
        run: ./deploy.sh
        env:
          DEPLOY_TOKEN: ${{ secrets.DEPLOY_TOKEN }}

4.9 Lỗi YAML Thường Gặp

❌ Indent sai (Tab thay vì Space)

❌ Sai — dùng tab hoặc indent lệch

server:
    host: localhost      # 4 spaces
  port: 3000             # 2 spaces → LỖI!
	debug: true            # Tab → LỖI!

✅ Đúng — nhất quán 2 spaces

server:
  host: localhost
  port: 3000
  debug: true

❌ Thiếu ngoặc kép cho ký tự đặc biệt

❌ Sai — ký tự đặc biệt không ngoặc

message: Error: file not found
url: https://example.com
regex: [a-z]+

✅ Đúng — bọc trong ngoặc kép

message: "Error: file not found"
url: "https://example.com"
regex: "[a-z]+"

❌ Boolean bất ngờ (Boolean Gotchas)

❌ Nguy hiểm — YAML tự chuyển thành boolean

country: NO        # → false (không phải "NO"!)
answer: yes        # → true (không phải "yes"!)
switch: on         # → true
version: 1.0       # → number 1 (mất .0!)

✅ An toàn — dùng ngoặc kép

country: "NO"      # → string "NO" ✓
answer: "yes"      # → string "yes" ✓
switch: "on"       # → string "on" ✓
version: "1.0"     # → string "1.0" ✓
⚠️

YAML tự chuyển yes, no, on, off, true, false thành boolean. Nếu muốn giữ là string, luôn bọc trong ngoặc kép!
