8.1 Markdown Là Gì?

Markdown là ngôn ngữ đánh dấu (markup language) siêu nhẹ — bạn viết text thuần với một vài ký hiệu đặc biệt, và nó sẽ được chuyển thành HTML đẹp mắt. File Markdown có đuôi .md.

Markdown source → Parser → HTML rendered

💡

Trong Claude Code, Markdown là NGÔN NGỮ CHÍNH để giao tiếp cấu hình. Hầu hết mọi file cấu hình quan trọng đều viết bằng Markdown.

Markdown được dùng ở đâu trong Claude Code?

8.9 CLAUDE.md Hoàn Chỉnh

Đây là ví dụ thực tế về file CLAUDE.md cho một dự án web — file quan trọng nhất khi dùng Claude Code:

# CLAUDE.md — Ví dụ hoàn chỉnh cho dự án e-commerce

# E-Commerce Platform — Claude Code Guidelines

## Tech Stack
- **Frontend**: React 18 + TypeScript + Vite
- **Backend**: Node.js + Express + TypeScript
- **Database**: PostgreSQL + Prisma ORM
- **Cache**: Redis
- **Testing**: Vitest (unit), Playwright (e2e)
- **Deployment**: Docker + AWS ECS

## Project Structure
```
src/
├── client/          # React frontend
│   ├── components/  # Reusable UI components
│   ├── pages/       # Route pages
│   ├── hooks/       # Custom React hooks
│   └── utils/       # Client utilities
├── server/          # Express backend
│   ├── routes/      # API endpoints
│   ├── services/    # Business logic
│   ├── middleware/   # Express middleware
│   └── models/      # Prisma models
└── shared/          # Shared types & constants
```

## Commands
- `npm run dev` — Start dev server (frontend + backend)
- `npm run build` — Build for production
- `npm test` — Run unit tests (Vitest)
- `npm run test:e2e` — Run e2e tests (Playwright)
- `npm run lint` — ESLint + Prettier check
- `npm run db:migrate` — Run Prisma migrations
- `npm run db:seed` — Seed database

## Coding Standards
- **Language**: Always use TypeScript, never plain JS
- **Components**: Functional components + hooks only
- **Naming**:
  - Components: `PascalCase` (e.g., `ProductCard.tsx`)
  - Utilities: `camelCase` (e.g., `formatPrice.ts`)
  - Constants: `UPPER_SNAKE_CASE`
  - Database tables: `snake_case`
- **Imports**: Use `@/` path alias for `src/`
- **Error handling**: Always use custom `AppError` class
- **API responses**: Follow `{ success, data, error }` format

## Rules
- NEVER commit `.env` files or secrets
- ALWAYS write tests for new features
- ALWAYS run `npm run lint` before committing
- Use Prisma for ALL database operations — no raw SQL
- API endpoints MUST have input validation (Zod)
- Components MUST be accessible (aria attributes)
- Max file length: 300 lines — split if longer

## Git Conventions
- Branch: `feature/xxx`, `fix/xxx`, `chore/xxx`
- Commit: `type(scope): message` (conventional commits)
- Examples:
  - `feat(cart): add coupon code support`
  - `fix(auth): handle expired tokens`
  - `chore(deps): update React to v18.3`
💡

CLAUDE.md là "bộ não" của Claude Code trong dự án. Claude sẽ đọc file này ĐẦU TIÊN khi bắt đầu — càng chi tiết và rõ ràng, Claude càng viết code đúng chuẩn dự án.

📌

Vị trí CLAUDE.md: Đặt ở root dự án cho quy tắc chung. Tạo thêm ~/.claude/CLAUDE.md cho quy tắc cá nhân áp dụng mọi dự án. Có thể đặt thêm CLAUDE.md trong thư mục con để override.

8.10 Slash Command Example

Slash commands là file .md trong .claude/commands/ — Claude Code đọc như template prompt:

# File: .claude/commands/review.md

Hãy review code theo các tiêu chí sau:

## Checklist
1. **Security** — Kiểm tra SQL injection, XSS, CSRF
2. **Performance** — N+1 queries, memory leaks, unnecessary re-renders
3. **Types** — TypeScript types đầy đủ, không dùng `any`
4. **Error Handling** — Try-catch đúng chỗ, error messages rõ ràng
5. **Tests** — Unit test cover happy path + edge cases
6. **Naming** — Biến/hàm có tên rõ nghĩa, theo convention

## Output Format
Trả về kết quả theo format:

### ✅ Tốt
- (liệt kê điểm tốt)

### ⚠️ Cần Sửa
- (liệt kê vấn đề + gợi ý fix)

### 📊 Điểm
- Security: X/10
- Performance: X/10
- Code Quality: X/10
# File: .claude/commands/create-api.md
# Slash command có tham số dùng $ARGUMENTS

Tạo API endpoint mới cho resource: $ARGUMENTS

## Yêu cầu
- Route file trong `src/server/routes/`
- Service file trong `src/server/services/`
- Zod validation schema

8.11 SKILL.md Example

SKILL.md định nghĩa một "kỹ năng" cho Claude Code agents, có YAML frontmatter ở đầu file:

# File: .claude/skills/database-migration.md

---
name: database-migration
description: Tạo và chạy database migration với Prisma
triggers:
  - "tạo migration"
  - "thay đổi database"
  - "thêm column"
  - "thêm table"
---

# Database Migration Skill

## Bước thực hiện

1. **Phân tích yêu cầu** — Xác định thay đổi cần thiết
2. **Sửa Prisma schema** — Edit `prisma/schema.prisma`
3. **Tạo migration** — Chạy `npx prisma migrate dev --name <name>`
4. **Verify** — Kiểm tra migration SQL đã generate
5. **Update types** — Chạy `npx prisma generate`
6. **Update seed** — Cập nhật `prisma/seed.ts` nếu cần

## Quy tắc

- Migration name: `snake_case`, mô tả ngắn gọn
  - Ví dụ: `add_coupon_codes_table`, `add_email_to_orders`
- KHÔNG BAO GIỜ sửa migration đã chạy
- Luôn thêm default value cho column mới trên table có data
- Dùng `@map` và `@@map` để giữ snake_case trong DB

## Prisma Schema Conventions

```prisma
model Product {
  id          String   @id @default(cuid())
  name        String
  price       Decimal  @db.Decimal(10, 2)
  description String?
  isActive    Boolean  @default(true)
  createdAt   DateTime @default(now()) @map("created_at")
  updatedAt   DateTime @updatedAt @map("updated_at")

  category    Category @relation(fields: [categoryId], references: [id])
  categoryId  String   @map("category_id")

  @@map("products")
}
```
📌

YAML Frontmatter nằm giữa hai dấu --- ở đầu file. Chứa metadata: tên, mô tả, triggers. Phần còn lại là nội dung Markdown bình thường mô tả kỹ năng.
