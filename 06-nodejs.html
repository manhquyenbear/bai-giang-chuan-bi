🟢 Node.js & npm

Nền tảng runtime JavaScript — công cụ cần thiết để cài đặt và chạy Claude Code, MCP servers

6.1 Node.js Là Gì?

Node.js là một JavaScript runtime — cho phép bạn chạy code JavaScript bên ngoài trình duyệt. Trước khi có Node.js, JavaScript chỉ chạy được trong Chrome, Firefox, Safari. Giờ đây, JS chạy được ở mọi nơi:

JavaScript chạy ở mọi nơi nhờ Node.js — từ trình duyệt, server, đến Claude Code

💡

Claude Code được viết bằng TypeScript/JavaScript và cài qua npm. Hiểu Node.js & npm = hiểu cách Claude Code hoạt động bên dưới.

Khái niệmGiải thích
Node.jsRuntime chạy JavaScript trên máy tính (dựa trên engine V8 của Chrome)
npmNode Package Manager — trình quản lý thư viện/package cho Node.js
npxCông cụ chạy package mà không cần cài đặt (đi kèm npm)
package.jsonFile cấu hình project — liệt kê dependencies, scripts, metadata
node_modulesThư mục chứa tất cả package đã cài — KHÔNG commit vào Git

6.2 Cài Đặt Node.js

Windows

# Cách 1: Tải installer từ website
# Vào https://nodejs.org → tải bản LTS → cài đặt

# Cách 2: Dùng winget (Windows Package Manager)
$ winget install OpenJS.NodeJS.LTS

# Cách 3: Dùng Chocolatey
$ choco install nodejs-lts

macOS

# Dùng Homebrew (khuyến nghị)
$ brew install node

# Hoặc dùng nvm (quản lý nhiều phiên bản)
$ curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
$ nvm install --lts
$ nvm use --lts

Xác nhận cài đặt thành công

$ node --version
v22.15.0

$ npm --version
10.9.2

$ npx --version
10.9.2
📌

Luôn cài bản LTS (Long Term Support) — ổn định, được hỗ trợ lâu dài. Tránh bản "Current" trừ khi cần tính năng mới nhất.

💡

Nếu dùng nvm (Node Version Manager), bạn có thể chuyển đổi phiên bản Node.js dễ dàng: nvm install 20, nvm use 18. Rất hữu ích khi làm nhiều project yêu cầu phiên bản khác nhau.

6.3 npm Là Gì?

npm (Node Package Manager) là kho thư viện lớn nhất thế giới với hơn 2 triệu package. Nó giúp bạn cài đặt, quản lý và chia sẻ code JavaScript.

LệnhChức năngVí dụ
npm init -yTạo project mới (file package.json)npm init -y
npm install <pkg>Cài package vào project (local)npm install express
npm install -g <pkg>Cài package toàn hệ thống (global)npm install -g @anthropic-ai/claude-code
npm installCài tất cả dependencies từ package.jsonnpm install
npm run <script>Chạy script định nghĩa trong package.jsonnpm run dev
npm testChạy test (tắt = npm run test)npm test
npm listXem danh sách package đã càinpm list --depth=0
npm outdatedKiểm tra package cần cập nhậtnpm outdated
npm uninstall <pkg>Gỡ packagenpm uninstall lodash
npm updateCập nhật tất cả packagenpm update
💡

Viết tắt: npm i = npm install, npm i -D = npm install --save-dev (cài cho dev only).

6.4 Tạo Project Node.js

Bước 1: Khởi tạo project

# Tạo thư mục project
$ mkdir my-node-app && cd my-node-app

# Khởi tạo package.json (flag -y = dùng mặc định)
$ npm init -y

Bước 2: File package.json được tạo

{
  "name": "my-node-app",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC"
}

Bước 3: Cài đặt dependencies

# Cài dependency chính (cho production)
$ npm install express dotenv

# Cài devDependency (chỉ dùng khi phát triển)
$ npm install -D nodemon eslint jest

Bước 4: Cấu trúc thư mục sau khi cài

my-node-app/
├── node_modules/        # 📦 Tất cả package (KHÔNG commit!)
│   ├── express/
│   ├── dotenv/
│   └── ... (hàng trăm thư mục con)
├── package.json         # 📋 Danh sách dependencies + config
├── package-lock.json    # 🔒 Phiên bản chính xác (PHẢI commit)
└── index.js             # 📄 File code chính
⚠️

KHÔNG BAO GIỜ commit node_modules/ vào Git! Luôn thêm vào .gitignore. Thư mục này có thể chứa hàng chục ngàn file và rất nặng.

# Tạo .gitignore
$ echo "node_modules/" > .gitignore

package.json sau khi cài dependencies

{
  "name": "my-node-app",
  "version": "1.0.0",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "dependencies": {
    "express": "^4.21.2",
    "dotenv": "^16.4.7"
  },
  "devDependencies": {
    "nodemon": "^3.1.9",
    "eslint": "^9.18.0",
    "jest": "^29.7.0"
  }
}
📌

Ký hiệu phiên bản: ^4.21.2 nghĩa là chấp nhận cập nhật minor/patch (4.x.x) nhưng không lên 5.x.x. ~4.21.2 chỉ cập nhật patch (4.21.x). Phiên bản chính xác: 4.21.2 (không có ký hiệu).

6.5 npm Scripts

Scripts là các lệnh tắt định nghĩa trong package.json — giúp chuẩn hóa cách chạy project cho cả team.

{
  "scripts": {
    "start": "node server.js",
    "dev": "nodemon server.js",
    "build": "tsc && node dist/index.js",
    "test": "jest --coverage",
    "lint": "eslint src/",
    "format": "prettier --write src/"
  }
}
# Chạy script
$ npm start           # Chạy "start" (không cần "run")
$ npm test            # Chạy "test" (không cần "run")
$ npm run dev         # Chạy "dev" (CẦN "run")
$ npm run build       # Chạy "build" (CẦN "run")
$ npm run lint        # Chạy "lint" (CẦN "run")
💡

Chỉ start và test là chạy trực tiếp (npm start, npm test). Tất cả script khác phải dùng npm run <tên>.

ScriptMục đích thường dùngLệnh chạy
startChạy app ở productionnpm start
devChạy app ở chế độ phát triển (auto-reload)npm run dev
buildBuild code (compile TypeScript, bundle, ...)npm run build
testChạy test suitenpm test
lintKiểm tra style codenpm run lint

6.6 npx — Chạy Package Không Cần Cài

npx cho phép bạn chạy một package npm trực tiếp mà không cần cài đặt trước. Package được tải về, chạy, rồi xóa. Đây là cách MCP servers hoạt động!

# So sánh: cài rồi chạy vs npx

# Cách cũ: cài global → chạy
$ npm install -g create-react-app
$ create-react-app my-app

# Cách mới: npx chạy luôn (không cần cài)
$ npx create-react-app my-app

# Thêm ví dụ npx
$ npx cowsay "Hello from npx!"     # Chạy thử package vui
$ npx http-server                   # Tạo web server nhanh
$ npx json-server db.json           # Fake API server

MCP Servers dùng npx

Đây là lý do chính bạn cần hiểu npx — MCP servers (Model Context Protocol) cho Claude Code đều được chạy qua npx:

# Thêm MCP server cho Claude Code
$ claude mcp add github -- npx -y @modelcontextprotocol/server-github
$ claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /path/to/dir
$ claude mcp add puppeteer -- npx -y @modelcontextprotocol/server-puppeteer

# Flag -y = tự động xác nhận cài đặt (không hỏi yes/no)

Luồng hoạt động: Claude Code → npx → MCP Server → Kết quả

💡

npx -y nghĩa là tự động đồng ý cài package. Không có -y, npx sẽ hỏi xác nhận mỗi lần — gây gián đoạn khi Claude Code tự động chạy MCP servers.

6.7 Global vs Local Install

Local install = cho project cụ thể / Global install = CLI tools dùng mọi nơi

Tiêu chíLocal npm install pkgGlobal npm install -g pkg
Lưu ở đâu./node_modules/Thư mục global của npm
Ghi vàopackage.jsonKhông ghi vào file nào
Phạm viChỉ project hiện tạiToàn bộ hệ thống
Chạy bằngnpx pkg hoặc npm scriptsGõ trực tiếp: claude
Dùng choLibraries, frameworks (express, react)CLI tools (claude-code, npm, eslint)
Ví dụnpm install expressnpm install -g @anthropic-ai/claude-code
💡

Nguyên tắc: Luôn cài local trừ khi package là CLI tool bạn cần gõ trực tiếp từ terminal. Cài global = mọi project dùng chung 1 phiên bản → dễ xung đột.

# Xem package global đã cài
$ npm list -g --depth=0
/usr/lib
├── @anthropic-ai/claude-code@1.0.12
├── npm@10.9.2
└── typescript@5.7.3

# Xem đường dẫn thư mục global
$ npm root -g

6.8 Cài Đặt Claude Code

Claude Code là CLI tool → cài global:

# Cài đặt Claude Code
$ npm install -g @anthropic-ai/claude-code

# Xác nhận cài đặt
$ claude --version
claude-code v1.0.12

# Cập nhật lên phiên bản mới nhất
$ npm update -g @anthropic-ai/claude-code

# Nếu cần cài lại hoàn toàn
$ npm uninstall -g @anthropic-ai/claude-code
$ npm install -g @anthropic-ai/claude-code
💡

Sau khi cài, gõ claude ở bất kỳ thư mục nào để bắt đầu. Claude Code sẽ yêu cầu đăng nhập Anthropic nếu chưa có API key.

📌

Trên macOS/Linux, nếu gặp lỗi quyền khi cài global, xem phần 6.9 bên dưới. KHÔNG dùng sudo npm install -g — có cách an toàn hơn.

6.9 Xử Lý Lỗi npm Thường Gặp

1. EACCES — Lỗi quyền truy cập (macOS/Linux)

# Lỗi:
npm ERR! code EACCES
npm ERR! Error: EACCES: permission denied, access '/usr/lib/node_modules'

# ❌ SAI — KHÔNG dùng sudo
$ sudo npm install -g claude-code

# ✅ ĐÚNG — Đổi thư mục global của npm
$ mkdir ~/.npm-global
$ npm config set prefix '~/.npm-global'
$ echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
$ source ~/.bashrc

# Giờ cài lại bình thường
$ npm install -g @anthropic-ai/claude-code
⚠️

Tuyệt đối KHÔNG dùng sudo npm install -g — điều này thay đổi quyền sở hữu file hệ thống và gây lỗi dây chuyền sau này.

2. Module not found

# Lỗi:
Error: Cannot find module 'express'

# Nguyên nhân: chưa cài dependencies
# Sửa: chạy npm install trong thư mục project
$ cd my-project
$ npm install

# Nếu vẫn lỗi, xóa node_modules và cài lại
$ rm -rf node_modules package-lock.json
$ npm install

3. Version conflict — Xung đột phiên bản

# Lỗi:
npm ERR! ERESOLVE unable to resolve dependency tree
npm ERR! peer dependency conflict

# Sửa: dùng flag --legacy-peer-deps
$ npm install --legacy-peer-deps

# Hoặc xem chi tiết xung đột
$ npm ls --all

4. Corrupted node_modules — "Nuclear option"

# Khi mọi cách đều thất bại — xóa sạch và cài lại
$ rm -rf node_modules
$ rm package-lock.json
$ npm cache clean --force
$ npm install

# Kiểm tra cache
$ npm cache verify
💡

Luôn thử npm install trước. Chỉ xóa node_modules khi thực sự cần thiết — việc cài lại có thể mất vài phút với project lớn.

5. node --version sai hoặc không tìm thấy

# Lỗi:
'node' is not recognized as an internal or external command

# Windows: Kiểm tra PATH
$ echo %PATH%
# Đảm bảo có C:\Program Files\nodejs\ trong PATH

# macOS/Linux: Kiểm tra PATH
$ echo $PATH
$ which node
# Nếu dùng nvm:
$ nvm use --lts
