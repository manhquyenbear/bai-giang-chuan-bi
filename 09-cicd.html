🔄 CI/CD & GitHub Actions

Tự động hóa kiểm thử, build và deploy — nền tảng DevOps hiện đại

9.1 CI/CD Là Gì?

CI/CD (Continuous Integration / Continuous Delivery) là quy trình tự động hóa việc kiểm thử, build và triển khai code mỗi khi có thay đổi.

Nguồn: docs.github.com — Understanding GitHub Actions

Pipeline CI/CD với GitHub Actions — Event kích hoạt Workflow → Jobs → Steps chạy trên Runner

💡

CI = "Mỗi lần push code, tự động chạy test để phát hiện lỗi sớm."
CD = "Nếu test pass, tự động deploy lên server."

❌ Không có CI/CD

# Dev push code lên main
# "Chạy ngon trên máy mình" 🤷
# Deploy thủ công bằng FTP
# Phát hiện bug trên production
# Rollback thủ công... hoảng loạn

✅ Có CI/CD

# Dev tạo PR
# CI tự động chạy test → lỗi ở dòng 42
# Dev sửa, push lại → test pass ✅
# Merge → CD tự deploy staging
# Approve → CD deploy production 🚀

9.2 GitHub Actions — CI/CD Của GitHub

GitHub Actions là nền tảng CI/CD tích hợp sẵn trong GitHub — không cần setup server riêng. Mọi thứ nằm trong thư mục .github/workflows/.

my-project/
├── .github/
│   └── workflows/
│       ├── ci.yml          # Chạy test khi push/PR
│       ├── deploy.yml      # Deploy khi merge vào main
│       └── review.yml      # Claude Code review PR
├── src/
├── tests/
└── package.json
Nguồn: docs.github.com — Understanding GitHub Actions

Cấu trúc GitHub Actions: Workflow → Jobs → Steps — mỗi Job chạy trên Runner riêng

Khái niệmGiải thíchVí dụ
WorkflowFile YAML định nghĩa pipeline.github/workflows/ci.yml
EventSự kiện kích hoạt workflowpush, pull_request, schedule
JobNhóm các bước chạy trên cùng runnertest, build, deploy
StepMột bước đơn lẻ trong jobChạy script, dùng action
ActionModule tái sử dụng (từ Marketplace)actions/checkout@v4
RunnerMáy ảo chạy jobubuntu-latest, windows-latest

9.3 Viết Workflow Đầu Tiên

Tạo file .github/workflows/ci.yml:

# .github/workflows/ci.yml
name: CI                          # Tên workflow (hiển thị trên GitHub)

on:                               # Khi nào chạy?
  push:
    branches: [main]              # Push vào main
  pull_request:
    branches: [main]              # Tạo PR vào main

jobs:                             # Danh sách jobs
  test:                           # Tên job
    runs-on: ubuntu-latest        # Chạy trên Ubuntu

    steps:                        # Các bước trong job
      - name: Checkout code       # Bước 1: Lấy code
        uses: actions/checkout@v4

      - name: Setup Node.js       # Bước 2: Cài Node
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install dependencies # Bước 3: Cài packages
        run: npm ci

      - name: Run tests            # Bước 4: Chạy test
        run: npm test

      - name: Run linter            # Bước 5: Kiểm tra code style
        run: npm run lint
📌

on: — Định nghĩa event trigger. Có thể là push, pull_request, schedule (cron), workflow_dispatch (chạy tay), v.v.

📌

jobs: — Mỗi job chạy trên một runner riêng (máy ảo mới). Các job chạy song song mặc định. Dùng needs: để chạy tuần tự.

📌

steps: — Mỗi step dùng uses: (action có sẵn) hoặc run: (chạy lệnh shell). Step chạy tuần tự trong job.

9.4 Ví Dụ Workflow Thực Tế

CI/CD cho Node.js — Test, Build, Deploy

# .github/workflows/full-pipeline.yml
name: Full CI/CD Pipeline

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [18, 20, 22]   # Test trên 3 phiên bản Node

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
      - run: npm ci
      - run: npm test

  build:
    needs: test                       # Chờ test pass mới build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npm run build
      - uses: actions/upload-artifact@v4
        with:
          name: build-output
          path: dist/

  deploy:
    needs: build                      # Chờ build xong mới deploy
    if: github.ref == 'refs/heads/main'  # Chỉ deploy từ main
    runs-on: ubuntu-latest
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: build-output
          path: dist/
      - name: Deploy to production
        run: |
          echo "Deploying to production..."
          # Thêm lệnh deploy thực tế ở đây

Claude Code Review — Tự động review PR

# .github/workflows/claude-review.yml
name: Claude Code Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pull-requests: write

    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0              # Lấy full history để diff

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Run Claude Review
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          claude -p "Review PR này. Tìm bugs, security issues, \
            và đề xuất cải thiện. Trả lời bằng tiếng Việt." \
            --output-format json > review.json

      - name: Post Review Comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const review = fs.readFileSync('review.json', 'utf8');
            await github.rest.issues.createComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              body: `## 🤖 Claude Code Review\n\n${review}`
            });

9.5 Secrets & Environment Variables

Secrets dùng để lưu thông tin nhạy cảm (API keys, tokens) — KHÔNG BAO GIỜ viết trực tiếp trong code.

Thêm Secret trên GitHub

# Đường dẫn trên GitHub:
Repository → Settings → Secrets and variables → Actions → New repository secret

# Ví dụ thêm:
Name:  ANTHROPIC_API_KEY
Value: sk-ant-xxxxxxxxxxxxx

Name:  DEPLOY_TOKEN
Value: ghp_xxxxxxxxxxxxx

Sử dụng trong workflow

# Dùng secret
env:
  ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

# Biến môi trường ở các cấp:

# 1. Cấp workflow (tất cả jobs đều thấy)
env:
  NODE_ENV: production

jobs:
  build:
    # 2. Cấp job (chỉ job này thấy)
    env:
      DATABASE_URL: ${{ secrets.DB_URL }}
    steps:
      - name: Deploy
        # 3. Cấp step (chỉ step này thấy)
        env:
          API_KEY: ${{ secrets.API_KEY }}
        run: echo "Deploying..."
⚠️

GitHub tự động mask secrets trong logs (hiện ***). Nhưng đừng dùng echo ${{ secrets.KEY }} — có thể bị lộ nếu secret chứa ký tự đặc biệt.

9.6 Các Actions Phổ Biến

ActionChức năngVí dụ
actions/checkout@v4Clone repo vào runnerBắt buộc trong hầu hết workflow
actions/setup-node@v4Cài đặt Node.jsnode-version: '20'
actions/setup-python@v5Cài đặt Pythonpython-version: '3.12'
actions/upload-artifact@v4Lưu file output giữa jobsLưu build output, test reports
actions/download-artifact@v4Tải artifact từ job trướcLấy build output để deploy
actions/cache@v4Cache dependenciesCache node_modules/

Ví dụ: Cache node_modules để tăng tốc

steps:
  - uses: actions/checkout@v4

  - name: Cache node_modules
    uses: actions/cache@v4
    with:
      path: node_modules
      key: ${{ runner.os }}-node-${{ hashFiles('package-lock.json') }}
      restore-keys: |
        ${{ runner.os }}-node-

  - name: Install dependencies
    run: npm ci

  - name: Run tests
    run: npm test
💡

Cache giúp giảm thời gian CI từ ~2 phút xuống ~30 giây bằng cách lưu lại node_modules/ giữa các lần chạy. Key dựa trên package-lock.json — thay đổi dependencies = cache mới.

Khi có CI/CD, mỗi PR và commit sẽ hiển thị trạng thái trực tiếp trên GitHub.

✅ CI Pass

All checks have passed
✅ test (18) — 45s
✅ test (20) — 42s
✅ test (22) — 43s
✅ build — 1m 12s
✅ deploy — 30s

→ "Merge pull request" button XANH

❌ CI Fail

Some checks were not successful
✅ test (18) — 45s
❌ test (20) — 38s  ← FAIL

⏭️ build — skipped
⏭️ deploy — skipped

9.8 Tích Hợp Claude Code Trong CI/CD

Claude Code có thể chạy trong CI/CD với flag -p (print mode — không cần terminal tương tác).

sequenceDiagram participant Dev as 👨‍💻 Developer participant GH as 🐙 GitHub participant CI as ⚙️ GitHub Actions participant Claude as 🤖 Claude Code Dev->>GH: Tạo Pull Request GH->>CI: Trigger workflow CI->>CI: Checkout code CI->>Claude: claude -p "Review PR này" Claude->>Claude: Phân tích code changes Claude-->>CI: Kết quả review (JSON) CI->>GH: Post comment lên PR GH-->>Dev: 🔔 Thông báo review

Luồng tự động: Push PR → Claude review → Comment kết quả

Workflow thực tế: Claude Code review tự động

# .github/workflows/claude-auto-review.yml
name: Claude Auto Review

on:
  pull_request:
    types: [opened, synchronize]

permissions:
  contents: read
  pull-requests: write

jobs:
  claude-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Get changed files
        id: diff
        run: |
          echo "files=$(git diff --name-only origin/main...HEAD | tr '\n' ' ')" \
            >> $GITHUB_OUTPUT

      - name: Install Claude Code
        run: npm install -g @anthropic-ai/claude-code

      - name: Review with Claude
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          claude -p "Hãy review các file sau: ${{ steps.diff.outputs.files }}
          Tập trung vào:
          1. Bugs tiềm ẩn
          2. Security issues
          3. Performance
          4. Code style
          Trả lời bằng tiếng Việt, format Markdown." > review.md

      - name: Comment on PR
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const review = fs.readFileSync('review.md', 'utf8');
            await github.rest.issues.createComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              body: `## 🤖 Claude Code Auto Review\n\n${review}`
            });
💡

claude -p = print mode — Claude nhận prompt, xử lý, trả kết quả về stdout. Hoàn hảo cho CI/CD vì không cần terminal tương tác.
