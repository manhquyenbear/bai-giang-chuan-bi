💻 Terminal / Command Line

Thành thạo dòng lệnh — nền tảng quan trọng nhất để dùng Claude Code

1.1 Terminal Là Gì?

Terminal (Command Line / CLI) là giao diện cho phép bạn giao tiếp với máy tính bằng cách gõ lệnh thay vì click chuột.

GUI vs CLI — hai cách giao tiếp với máy tính

💡

Claude Code chạy 100% trên terminal — không có giao diện đồ họa. Thiếu kỹ năng CLI = không dùng được Claude Code.

Mở Terminal trên mỗi OS

OSCách mởTerminal mặc định
WindowsWin + R → gõ cmdPowerShell / CMD
macOSCmd + Space → "Terminal"Terminal.app
LinuxCtrl + Alt + TBash
📌

Windows: Nên cài Windows Terminal và dùng Git Bash (cài kèm Git) để có trải nghiệm giống Linux/Mac.

1.3 Lệnh Di Chuyển & Xem Thư Mục

pwd — Đang ở đâu?

# Print Working Directory
$ pwd
/Users/qudomanh/Documents/project

ls — Liệt kê nội dung

# Cơ bản
$ ls
README.md  package.json  src/  node_modules/

# Chi tiết (-l) + file ẩn (-a)
$ ls -la
drwxr-xr-x  .git/
-rw-r--r--  .gitignore
-rw-r--r--  README.md
-rw-r--r--  package.json
drwxr-xr-x  src/

cd — Di chuyển

$ cd src          # Vào thư mục con
$ cd ..           # Lên thư mục cha
$ cd ~            # Về home
$ cd -            # Quay lại thư mục vừa rời
$ cd ~/Documents/project/src  # Đi theo đường dẫn tuyệt đối

1.4 Tạo, Xóa, Copy, Di Chuyển

# Tạo thư mục (kể cả thư mục cha)
$ mkdir -p src/components/buttons

# Tạo file rỗng
$ touch index.js App.js

# Copy file / thư mục (-r = recursive)
$ cp config.js config.backup.js
$ cp -r src/ src-backup/

# Di chuyển / đổi tên
$ mv old-name.js new-name.js
$ mv config.js src/config.js

# Xóa (CẨN THẬN — không có thùng rác!)
$ rm temp.js
$ rm -r old-backup/
⚠️

rm xóa vĩnh viễn — KHÔNG có thùng rác! Không bao giờ chạy rm -rf / hay rm -rf * nếu không chắc chắn.

Ví dụ: Tạo cấu trúc project mới

$ mkdir my-web-app && cd my-web-app
$ mkdir -p src/{components,utils,styles} tests docs
$ touch src/index.js src/App.js README.md .gitignore
$ ls -R

1.6 Pipe | và Redirect >

Đây là tính năng cực kỳ mạnh — nối output lệnh này thành input lệnh kia.

ls src/ | grep ".js" | wc -l → liệt kê → lọc .js → đếm dòng = 5

# Pipe | — nối lệnh
$ ls src/ | grep ".js" | wc -l      # Đếm file .js
$ cat log.txt | grep "ERROR"         # Lọc dòng lỗi
$ ps aux | grep "node"               # Tìm process node

# Redirect > — ghi output ra file
$ echo "Hello" > greeting.txt        # GHI ĐÈ
$ echo "Line 2" >> greeting.txt      # THÊM vào cuối
$ ls -la > file-list.txt             # Lưu danh sách file
$ npm test > results.txt 2>&1        # Lưu cả output + error

1.7 Tìm Kiếm File & Nội Dung

find — Tìm file theo tên

$ find . -name "*.js"
$ find . -type d -name "src"
$ find . -mtime -2 -name "*.js"

grep — Tìm nội dung trong file

$ grep -r "TODO" src/
$ grep -n "import" App.js
$ grep -i "error" server.log
# Ví dụ: phân tích log lỗi
$ grep "ERROR" server.log | wc -l            # Đếm số lỗi
$ grep "ERROR" server.log > errors-only.log   # Lưu lỗi ra file
$ grep "ERROR" server.log | grep -i "database" # Lỗi liên quan DB

1.8 Biến Môi Trường

Biến lưu ở cấp hệ thống, các chương trình có thể đọc được.

# Xem biến
$ echo $HOME         # /Users/qudomanh
$ echo $PATH         # Danh sách thư mục chứa chương trình

# Tạo biến
$ export API_KEY="sk-abc123"
$ echo $API_KEY      # sk-abc123

# QUAN TRỌNG cho Claude Code:
$ export GITHUB_TOKEN="ghp_xxxxxxxxxxxx"
$ export ANTHROPIC_API_KEY="sk-ant-xxxxx"
💡

Biến export chỉ tồn tại trong session hiện tại. Để lưu vĩnh viễn, thêm vào file ~/.bashrc hoặc ~/.zshrc.
