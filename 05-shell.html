📜 Shell Scripting

Viết script Bash — tự động hóa mọi thứ và tạo hooks cho Claude Code

5.1 Shell Script Là Gì?

Shell script là một file text chứa các lệnh terminal, được thực thi tuần tự bởi shell (thường là Bash). Thay vì gõ từng lệnh một, bạn gom tất cả vào file .sh và chạy một lần.

Thủ công vs Script — tự động hóa giúp tiết kiệm thời gian và tránh sai sót

💡

Claude Code hooks chạy bằng shell script! Ví dụ: hook PreToolUse kiểm tra trước khi Claude thực thi lệnh, hook PostToolUse chạy sau khi xong. Hiểu shell = viết hooks mạnh mẽ.

5.2 Tạo Script Đầu Tiên

Bước 1: Tạo file với shebang

#!/bin/bash
# File: hello.sh — Script đầu tiên

echo "🎉 Xin chào! Bạn đang chạy script Bash đầu tiên."
echo "📅 Ngày hiện tại: $(date)"
echo "📂 Thư mục hiện tại: $(pwd)"
echo "👤 Người dùng: $USER"
📌

Shebang #!/bin/bash — dòng đầu tiên BẮT BUỘC có. Nó cho hệ thống biết dùng /bin/bash để chạy file này. Không có shebang = hệ thống không biết dùng trình thông dịch nào.

Bước 2: Cấp quyền thực thi

# Cấp quyền chạy (+x = executable)
$ chmod +x hello.sh

# Kiểm tra quyền
$ ls -l hello.sh
-rwxr-xr-x  1 qudomanh  staff  180 Apr 14 10:30 hello.sh
#  ^ x = có quyền execute

Bước 3: Chạy script

# Cách 1: Dùng ./ (khuyến nghị)
$ ./hello.sh

# Cách 2: Gọi trực tiếp bash
$ bash hello.sh

# Output:
🎉 Xin chào! Bạn đang chạy script Bash đầu tiên.
📅 Ngày hiện tại: Mon Apr 14 10:30:00 +07 2026
📂 Thư mục hiện tại: /Users/qudomanh/project
👤 Người dùng: qudomanh

✅ Cách đúng

#!/bin/bash
chmod +x script.sh
./script.sh

❌ Lỗi thường gặp

# Thiếu shebang
# Quên chmod +x
# Chạy: script.sh (thiếu ./)

5.3 Biến Trong Bash

Khai báo biến

#!/bin/bash

# KHÔNG có khoảng trắng quanh dấu =
name="Claude Code"
version=3
project_dir="/home/user/project"

echo "Tool: $name"
echo "Version: $version"
echo "Dir: $project_dir"
⚠️

KHÔNG có khoảng trắng quanh = — đây là lỗi phổ biến nhất! name = "Claude" sẽ BÁO LỖI. Phải viết name="Claude".

✅ Đúng

name="Claude"
count=10
path="/usr/bin"

❌ Sai — có khoảng trắng

name = "Claude"   # Lỗi!
count =10         # Lỗi!
path= "/usr/bin"  # Lỗi!

Biến đặc biệt

BiếnÝ nghĩaVí dụ
$0Tên script đang chạy./deploy.sh → $0 = ./deploy.sh
$1, $2, ...Tham số truyền vào./script.sh hello → $1 = hello
$#Số lượng tham số./script.sh a b c → $# = 3
$@Tất cả tham số./script.sh a b → $@ = a b
$?Exit code lệnh trước0 = thành công, khác 0 = lỗi
$$PID của script hiện tạiDùng tạo file tạm: /tmp/lock.$$

Command Substitution — $(...)

#!/bin/bash
# Chạy lệnh và lấy kết quả vào biến

current_date=$(date +%Y-%m-%d)
file_count=$(ls src/*.js 2>/dev/null | wc -l)
git_branch=$(git branch --show-current)
disk_usage=$(df -h / | tail -1 | awk '{print $5}')

echo "📅 Ngày: $current_date"
echo "📁 Số file JS: $file_count"
echo "🔀 Branch: $git_branch"
echo "💾 Disk: $disk_usage"
💡

Luôn dùng $(...) thay vì backtick `...`. Cú pháp $(...) dễ đọc hơn, hỗ trợ lồng nhau, và là chuẩn hiện đại.

5.4 Điều Kiện if/else

Cú pháp cơ bản

#!/bin/bash

if [ điều_kiện ]; then
    # Đúng → chạy đây
    echo "Điều kiện đúng"
elif [ điều_kiện_khác ]; then
    # Điều kiện 2 đúng
    echo "Điều kiện 2"
else
    # Sai hết → chạy đây
    echo "Điều kiện sai"
fi
💡

BẮT BUỘC có khoảng trắng sau [ và trước ]. Viết [ -f file ] — KHÔNG phải [-f file].

Toán tử so sánh số

Toán tửÝ nghĩaVí dụ
-eqBằng (equal)[ $a -eq $b ]
-neKhác (not equal)[ $a -ne 0 ]
-gtLớn hơn (greater than)[ $a -gt 10 ]
-geLớn hơn hoặc bằng[ $a -ge 5 ]
-ltNhỏ hơn (less than)[ $a -lt 100 ]
-leNhỏ hơn hoặc bằng[ $a -le 50 ]

Toán tử so sánh chuỗi

Toán tửÝ nghĩaVí dụ
=Bằng[ "$str" = "hello" ]
!=Khác[ "$str" != "" ]
-zChuỗi rỗng[ -z "$str" ]
-nChuỗi KHÔNG rỗng[ -n "$str" ]

Kiểm tra file & thư mục

Toán tửÝ nghĩaVí dụ
-fFile tồn tại (regular file)[ -f package.json ]
-dThư mục tồn tại[ -d node_modules ]
-eTồn tại (file hoặc thư mục)[ -e .env ]
-xCó quyền thực thi[ -x deploy.sh ]
-rCó quyền đọc[ -r config.yml ]
-sFile tồn tại VÀ không rỗng[ -s output.log ]

Ví dụ thực tế: Pre-build check

#!/bin/bash
# pre-build-check.sh — Kiểm tra trước khi build project

echo "🔍 Kiểm tra môi trường build..."

# Kiểm tra Node.js
if ! command -v node &> /dev/null; then
    echo "❌ Node.js chưa cài đặt!"
    exit 1
fi

# Kiểm tra version Node.js >= 18
node_version=$(node -v | cut -d'v' -f2 | cut -d'.' -f1)
if [ "$node_version" -lt 18 ]; then
    echo "❌ Cần Node.js >= 18, hiện tại: v$node_version"
    exit 1
fi
echo "✅ Node.js v$node_version"

# Kiểm tra package.json
if [ ! -f "package.json" ]; then
    echo "❌ Không tìm thấy package.json — bạn có đang ở đúng thư mục?"
    exit 1
fi
echo "✅ package.json tồn tại"

# Kiểm tra node_modules
if [ ! -d "node_modules" ]; then
    echo "⚠️ Chưa cài dependencies, đang chạy npm install..."
    npm install
fi
echo "✅ Dependencies đã sẵn sàng"

# Kiểm tra biến môi trường
if [ -z "$API_KEY" ]; then
    echo "⚠️ Chưa set API_KEY — một số tính năng có thể không hoạt động"
fi

echo ""
echo "🎉 Tất cả kiểm tra OK — sẵn sàng build!"
npm run build

5.5 Vòng Lặp

Vòng lặp for

#!/bin/bash

# Lặp qua danh sách
for fruit in apple banana cherry; do
    echo "Quả: $fruit"
done

# Lặp qua file
for file in src/*.js; do
    echo "📄 File JS: $file"
done

# Lặp qua số (C-style)
for ((i=1; i<=5; i++)); do
    echo "Lần $i"
done

# Lặp qua tham số script
for arg in "$@"; do
    echo "Tham số: $arg"
done

Vòng lặp while

#!/bin/bash

# Đếm ngược
count=5
while [ $count -gt 0 ]; do
    echo "⏳ Còn $count giây..."
    sleep 1
    count=$((count - 1))
done
echo "🚀 Go!"

# Chờ service sẵn sàng
max_retries=10
retry=0
while [ $retry -lt $max_retries ]; do
    if curl -s http://localhost:3000/health > /dev/null 2>&1; then
        echo "✅ Service đã sẵn sàng!"
        break
    fi
    retry=$((retry + 1))
    echo "⏳ Đợi service... ($retry/$max_retries)"
    sleep 2
done

Đọc file theo từng dòng

#!/bin/bash
# Đọc file line by line — rất hữu ích

while IFS= read -r line; do
    echo "Dòng: $line"
done < config.txt

# Thực tế: Xử lý danh sách server
while IFS= read -r server; do
    # Bỏ qua dòng trống và comment
    [[ -z "$server" || "$server" == \#* ]] && continue
    echo "🔄 Deploying to: $server"
    ssh "$server" "cd /app && git pull && pm2 restart all"
done < servers.txt

Ví dụ thực tế: Backup configs

#!/bin/bash
# backup-configs.sh — Backup các file cấu hình quan trọng

backup_dir="$HOME/backups/$(date +%Y-%m-%d_%H%M%S)"
mkdir -p "$backup_dir"

# Danh sách file cần backup

    "$HOME/.bashrc"
    "$HOME/.gitconfig"
    "$HOME/.claude/settings.json"
    "$HOME/.ssh/config"
)

count=0
for file in "${config_files[@]}"; do
    if [ -f "$file" ]; then
        cp "$file" "$backup_dir/"
        echo "✅ Backup: $file"
        count=$((count + 1))
    else
        echo "⚠️ Không tìm thấy: $file"
    fi
done

echo ""
echo "📦 Đã backup $count file vào: $backup_dir"
ls -la "$backup_dir/"

5.6 Exit Code

Mỗi lệnh khi chạy xong đều trả về một exit code — con số cho biết lệnh thành công hay thất bại.

Exit CodeÝ nghĩaVí dụ
0✅ Thành côngls tìm thấy file
1❌ Lỗi chunggrep không tìm thấy
2❌ Sai cú pháp lệnhTham số không hợp lệ
126❌ Không có quyền thực thiQuên chmod +x
127❌ Không tìm thấy lệnhGõ sai tên lệnh
130❌ Bị Ctrl+CNgười dùng hủy
 
# Kiểm tra exit code bằng $?
$ ls /tmp
$ echo $?
0                # Thành công

$ ls /khong-ton-tai
$ echo $?
2                # Lỗi — không tìm thấy

# Dùng exit trong script để trả exit code
#!/bin/bash
if [ ! -f "package.json" ]; then
    echo "❌ Thiếu package.json"
    exit 1       # Thoát với mã lỗi
fi
echo "✅ OK"
exit 0           # Thoát thành công

Nối lệnh với && và ||

# && — chỉ chạy lệnh sau nếu lệnh trước THÀNH CÔNG (exit 0)
$ npm test && npm run build && echo "🎉 Build OK"

# || — chỉ chạy lệnh sau nếu lệnh trước THẤT BẠI (exit != 0)
$ npm test || echo "❌ Test failed"

# Kết hợp cả hai
$ npm test && echo "✅ Pass" || echo "❌ Fail"

# Thực tế: nếu thư mục chưa có thì tạo
$ [ -d "dist" ] || mkdir dist

Claude Code hooks dùng exit code để quyết định hành vi. Hook trả exit 0 = cho phép tiếp tục, exit 1 = chặn hành động. Đây là cơ chế kiểm soát quan trọng nhất!

5.7 Đọc Input Từ Stdin

Trong shell scripting, stdin (standard input) là luồng dữ liệu đầu vào. Bạn có thể đọc input từ người dùng hoặc từ pipe.

Đọc input cơ bản

#!/bin/bash

# Đọc từ người dùng
read -p "Tên bạn là gì? " name
echo "Xin chào, $name!"

# Đọc toàn bộ stdin vào biến
input=$(cat)
echo "Bạn đã nhập: $input"

# Đọc từ pipe
echo '{"name":"Claude"}' | ./process.sh

Claude Code hooks nhận JSON qua stdin

Khi Claude Code gọi hook, nó gửi JSON qua stdin chứa thông tin về hành động. Script đọc bằng input=$(cat).

Cách Claude Code hooks hoạt động — nhận JSON qua stdin, quyết định bằng exit code

Ví dụ: Claude Code hook chặn lệnh nguy hiểm

#!/bin/bash
# ~/.claude/hooks/pre-tool-check.sh
# Hook PreToolUse — kiểm tra trước khi Claude chạy lệnh

# Đọc JSON từ stdin
input=$(cat)

# Lấy tên tool và nội dung lệnh (dùng grep đơn giản)
tool=$(echo "$input" | grep -o '"tool_name":"[^"]*"' | cut -d'"' -f4)
command=$(echo "$input" | grep -o '"input":"[^"]*"' | cut -d'"' -f4)

# Danh sách lệnh nguy hiểm
dangerous_patterns=(
    "rm -rf /"
    "rm -rf ~"
    "drop database"
    "mkfs"
    "dd if="
    "> /dev/sda"
)

# Kiểm tra từng pattern
for pattern in "${dangerous_patterns[@]}"; do
    if echo "$command" | grep -qi "$pattern"; then
        echo "🚫 BLOCKED: Lệnh nguy hiểm '$pattern' detected!" >&2
        exit 2    # exit 2 = chặn hành động
    fi
done

# Lệnh an toàn — cho phép
exit 0
💡

Hook dùng exit 2 để chặn hành động và gửi lỗi qua stderr (dùng >&2). exit 0 = cho phép. Đây là giao thức chuẩn của Claude Code hooks.

5.8 Functions (Hàm)

Hàm giúp tái sử dụng code và tổ chức script gọn gàng hơn.

Cú pháp

#!/bin/bash

# Cách 1: Dùng function keyword
function greet() {
    echo "Xin chào, $1!"
}

# Cách 2: Không cần keyword (khuyến nghị — tương thích POSIX)
greet() {
    echo "Xin chào, $1!"
}

# Gọi hàm
greet "Claude"    # Output: Xin chào, Claude!
greet "User"      # Output: Xin chào, User!

Hàm log — in có màu và timestamp

#!/bin/bash

# Hàm log tiện ích — dùng lại trong mọi script
log_info()    { echo "[$(date '+%H:%M:%S')] ℹ️  $*"; }
log_success() { echo "[$(date '+%H:%M:%S')] ✅ $*"; }
log_warning() { echo "[$(date '+%H:%M:%S')] ⚠️  $*" >&2; }
log_error()   { echo "[$(date '+%H:%M:%S')] ❌ $*" >&2; }

# Sử dụng
log_info "Bắt đầu deploy..."
log_success "Build xong"
log_warning "Cache hết hạn"
log_error "Database không phản hồi"

Hàm check_dependency

#!/bin/bash

# Kiểm tra tool có cài chưa
check_dependency() {
    local cmd="$1"
    local name="${2:-$1}"    # Tên hiển thị, mặc định = tên lệnh

    if command -v "$cmd" &> /dev/null; then
        log_success "$name đã cài: $(command -v "$cmd")"
        return 0
    else
        log_error "$name chưa cài! Vui lòng cài đặt trước."
        return 1
    fi
}

# Sử dụng
check_dependency "node" "Node.js"   || exit 1
check_dependency "git" "Git"        || exit 1
check_dependency "npm" "npm"        || exit 1
check_dependency "docker" "Docker"  || log_warning "Docker không có — skip container tests"

Ví dụ thực tế: Deploy script hoàn chỉnh

#!/bin/bash
# deploy.sh — Script deploy hoàn chỉnh
set -euo pipefail    # Dừng ngay khi có lỗi

# ─── Config ───────────────────────────────────
PROJECT_DIR="$(cd "$(dirname "$0")" && pwd)"
DEPLOY_BRANCH="main"
BUILD_DIR="dist"

# ─── Functions ────────────────────────────────
log_info()    { echo "[$(date '+%H:%M:%S')] ℹ️  $*"; }
log_success() { echo "[$(date '+%H:%M:%S')] ✅ $*"; }
log_error()   { echo "[$(date '+%H:%M:%S')] ❌ $*" >&2; }

check_dependency() {
    command -v "$1" &> /dev/null || {
        log_error "$1 chưa cài đặt!"
        exit 1
    }
}

cleanup() {
    log_info "Dọn dẹp file tạm..."
    rm -f /tmp/deploy_lock_$$
}
trap cleanup EXIT    # Luôn cleanup khi script kết thúc

# ─── Main ─────────────────────────────────────
cd "$PROJECT_DIR"
log_info "🚀 Bắt đầu deploy từ $PROJECT_DIR"

# 1. Kiểm tra dependencies
check_dependency "node"
check_dependency "npm"
check_dependency "git"

# 2. Kiểm tra branch
current_branch=$(git branch --show-current)
if [ "$current_branch" != "$DEPLOY_BRANCH" ]; then
    log_error "Phải ở branch '$DEPLOY_BRANCH', hiện tại: '$current_branch'"
    exit 1
fi
log_success "Branch: $current_branch"

# 3. Kiểm tra uncommitted changes
if [ -n "$(git status --porcelain)" ]; then
    log_error "Có thay đổi chưa commit! Commit trước khi deploy."
    exit 1
fi
log_success "Working directory sạch"

# 4. Cài dependencies & build
log_info "Cài dependencies..."
npm ci --silent

log_info "Chạy tests..."
npm test || { log_error "Tests failed!"; exit 1; }

log_info "Build project..."
npm run build || { log_error "Build failed!"; exit 1; }

# 5. Kiểm tra build output
if [ ! -d "$BUILD_DIR" ]; then
    log_error "Thư mục $BUILD_DIR không tồn tại sau build!"
    exit 1
fi

file_count=$(find "$BUILD_DIR" -type f | wc -l)
log_success "Build xong — $file_count files trong $BUILD_DIR/"

log_info ""
log_success "🎉 Deploy hoàn tất!"
📌

set -euo pipefail — dòng "thần chú" nên có trong mọi script: -e dừng khi lỗi, -u báo lỗi biến chưa khai báo, -o pipefail bắt lỗi trong pipe.
